📊 PIPELINE ETL EM R – Sistema de Ingestão, Padronização, Limpeza e Validação de Dados

Pipeline de engenharia de dados desenvolvido em R para ingestão, padronização, limpeza e validação de dados de acidentes de trânsito da cidade de Recife (2019).

🛠️ Tecnologias
R yaml dplyr logger stringr lubridate glue hms readr RStudio R Markdown Git GitHub
⚙️ Funcionalidades
  • Ingestão de dados
    • Ler dados brutos (raw data) e garantir integridade estrutural mínima
  • Padronização
    • Padronização de nomes, conversão de tipos (data, hora) e normalização de variáveis textuais
  • Limpeza
    • Normalização de textos vazios e substituição de NA ou strings em variáveis numéricas quando aplicável
  • Validação
    • Validação de todo pipeline
📈 Resultados

Resumidamente, o fluxo coleta, padroniza, limpa e valida os dados e gera um arquivo CSV pronto para análise de dados ou implementação de Dashboard.

📊 DASHBOARD QLIKVIEW – Dashboard Estratégico de Mobilidade Urbana

Este projeto tem como objetivo analisar e visualizar dados de acidentes de trânsito, permitindo a identificação de padrões, tendências na ocorrência desses acidentes.

🛠️ Tecnologias
Qlikview 12 Google Documentos Google Planilhas PhotoShop Git GitHub
⚙️ Estrutura
  • Capa: visão geral do projeto e objetivos
  • Visão Geral: KPIs principais
  • Análise Temporal: evolução por dias, semanas, meses, trimestres e horas
  • Análise por veículos: tipos de veículos
  • Análise por condição das vias, sinalização e clima
📈 Resultados

O foco é transformar dados brutos em informação estratégica, de fácil compreensão para diferentes perfis de usuários.
